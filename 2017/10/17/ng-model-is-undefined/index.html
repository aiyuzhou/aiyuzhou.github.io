ng-model is undefined

作者 Annie Chou 日期 2017-10-17
ng-model angularjs angularjs作用域
ng-model is undefined

angular控制器controller里获取不到ng-model的值，获取为undefined

<div class="form-group">
    <input type="text" class="form-control" placeholder="Add Firmware Title" ng-model="afVersion">
</div>

在controller里打印afVersion：

console.log($scope.afVersion); //undefined

原因：所在scope不是同一个，导致取不到数据，AngularJS中的子指令执行的时候会创建一个新的子作用域，所以要先理清楚afVersion是在哪个作用域下

解决方案一：

确认ng-model之上有几层$scope将afVersion修改为$parent.afVersion/$parent.$parent.afVersion…

解决方案二：

######这段代码之上有两个子scope被创建，如果采用第一种解决方案，可写作$parent.$parent.afVersion，但是在实际项目中，我们经常会使用指令以及自定义指令，如果在很底层的scope中可能要加很多个$parent，所以究其本质，我们应该找到一种统一的方式来解决这类型问题。

本质问题是，如果我们在子作用域上查找一个属性，则会沿着原型链向上查找，但如果在子作用域内给父作用域的属性赋值，则会覆盖父作用域中的同名属性。

<div class="form-group">
    <input type="text" class="form-control" placeholder="Add Firmware Title" ng-model="afVersion">
    <span>{{afVersion}}</span>
</div>

console.log($scope.afVersion); //undefined
到这里就可以解释为什么在span里可以读到afVersion，但是$scope.afVersion打印出来是Undefined，ng-model=”afVersion”和在同一个子作用域下，$scope.afVersion在他们的父级的父级作用域，在input输入值的时候，其实是给childscope创建了新的afVersion属性值，console.log打印$scope.afVersion为Undefined，是因为父级该层scope上没有afVersion属性。

修改成以下方式：

$scope.modelData= {
    afVersion: ''
};
<input type="text" class="form-control" placeholder="Add Firmware Title" ng-model="modelData.afVersion">

input输入值的时候，在子作用域内找不到modelData对象，就会沿着原型链向上查找，然后更新afVersion属性值。

这里有AngularJS的scope详解：Understanding-Scopes，写的非常详细完整。
