前端基础问题总结

css相关问题

一、display：none和visibility：hidden的区别？

  • display：none 是隐藏对应的元素，在文档布局中不再给它分配空间
  • visibility：hidden 隐藏相应元素，但是文档流中位置保存

二、CSS中link和@import的区别？

  • link
    • 属于HTML标签
    • 页面加载时，link会同时被加载
    • 无兼容性问题
  • @import
    • 是CSS提供的标签
    • 在页面加载完成后才加载@import引用的CSS文件
    • 只有在IE5以上才可用，低于IE5不兼容
  • @import缺点，造成无样式内容闪烁(FOUC = flash of unstyled content)，因为浏览器会先加载DOM，然后才载入CSS样式表

三、IE盒模型和标准W3C盒模型

  • 标准W3C盒模型：
    • 包括margin、border、padding、content，并且content不包含其他部分
    • content的width = width

  • IE盒模型：
    • 也包括margin、border、padding、content，但是content包括border和padding
    • content的width = width + padding + border

四、CSS选择符有哪些？

  • id选择器(#myid)
  • 类选择器(.myclassname)
  • 标签选择器(div,h1,p)
  • 相邻选择器(h1 + p)
  • 子选择器(ul > li)
  • 后代选择器(li a)
  • 通配符选择器(*)
  • 属性选择器(a[rel = “external”])
  • 伪类选择器(a:hover,li:nth-child)

五、CSS可继承的属性

  • font-size
  • font-family
  • color
  • text-intent

六、CSS不可继承的属性

  • border
  • padding
  • margin
  • width
  • height

七、CSS优先级

! important > id > class > tag

八、CSS3新增的伪类

  • :first-child选择某个元素的第一个子元素；
  • :last-child选择某个元素的最后一个子元素；
  • :nth-child()选择某个元素的一个或多个特定的子元素；
  • :nth-last-child()选择某个元素的一个或多个特定的子元素，从这个元素的最后一个子元素开始算；
  • :nth-of-type()选择指定的元素；
  • :nth-last-of-type()选择指定的元素，从元素的最后一个开始计算；
  • :first-of-type选择一个上级元素下的第一个同类子元素；
  • :last-of-type选择一个上级元素的最后一个同类子元素；
  • :only-child选择的元素是它的父元素的唯一一个了元素；
  • :only-of-type选择一个元素是它的上级元素的唯一一个相同类型的子元素；
  • :empty选择的元素里面没有任何内容。

九、为什么要初始化CSS样式

因为浏览器的兼容问题，不同浏览器对有些标签的默认值是不同的，如果没有对CSS初始化，往往不同的浏览器之间的页面显示差异。

十、BFC:块级格式化上下文

  • 根元素
  • float属性不为none
  • position为absolute或fixed
  • display为inline-block,table-cell,table-caption,flex,inline-flex
  • overflow不为visible

在一个BFC的盒子内的子元素的样式不会影响到外面的元素，并且同一个BFC中的两个相邻的块级盒在垂直方向(和布局方向有关系)的margin会发生折叠。

十一、CSS sprite（CSS精灵）

图像精灵—是放入一张单独的图片中的一系列图像。
大量包含图像的网页需要更长的时间来下载，同时会生成多个服务器请求。
使用图像精灵将减少服务器请求数量并节约带宽。虽然请求可以并发，但是也有限制，一般浏览器都是6个。

我们将通过CSS的background-image,background-repeat,background-position的组合进行背景定位。

一、语义化的理解

  • 去掉或者丢失样式的时候能够让页面呈现出清晰的结构
  • 有利于SEO：有利于爬虫抓取更多的有效信息
  • 方便其他设备解析
  • 便于团队开发和维护：语义化更具有可读性

三、常见的浏览器兼容问题

  • 浏览器默认的margin和padding不同，解决方案是加一个全局初始化*{margin：0；padding：0}
  • IE6双边距：块元素设置float，又同时设置margin情况下，在IE6中margin值会加倍
    • 解决方法：对其加入_display:inline；
  • IE获取scrollY的值要通过document.documentElement.scrollY来获取
  • chrome中文界面默认将小于12px的文本强制按12px显示，可以通过CSS属性：-webkit-text-size-adjust：none；解决

三、浮动元素的工作原理，以及清除浮动元素的技巧

浮动元素：脱离文档流，不占据空间，浮动元素碰到包含它的边框或者浮动元素的边框则停留。

浮动元素的问题：

  • 父元素的高度无法撑开，影响父元素同级的元素
  • 与浮动元素同级的非浮动元素会跟随其后

清除浮动的方法：

  • 额外标签：在所有浮动元素后面增加空标签，并设置CSS clear：both；但是增加了无意义的标签
  • 父元素设置overflow：auto;zoom：1；zoom用来兼容IE6;要实现移除隐藏，就要计算什么时候会溢出；所以就会把子元素都计算进去，顺便达到撑开父元素的效果

  • 使用after伪对象：

    .clearfix:after {

    visibility: hidden;
display: block;
font-size: 0;
content: " ";
clear: both;
height: 0;

    }

新特性

  • 更好的语义化标签（header，nav，footer，aside，article，section，figure，figcaption等）
  • 音频，视频API
  • 画布 canvas API
  • 地理API
  • 本地离线存储localStorage持久化存储，sessionStorage在浏览器关闭后自动删除

五、iframe

优点：解决了加载缓慢的第三方内容，图标和广告等，并行加载脚本

缺点：iframe会阻塞主页面的onload事件，即使内容为空，加载也需要时间，没有语义

六、页面加载过程

浏览器解析 -> 解析HTML节点到DOM树 -> DOM树解析完成（触发DOMContentLoaded）-> 下载资源并解析资源（包括iframe）-> 资源全部下载并解析完毕 -> 页面加载完成（触发onload事件）

七、如何对网站的文件和资源进行优化？

  • 文件合并
  • 文件最小化、文件压缩
  • 使用cdn托管

八、减少页面加载时间的方法？

  • 优化图片
  • 图像格式的选择
  • 优化CSS
  • 减少http请求（合并文件，图片）

九、js延迟加载

  • defer：通过设置defer属性，可以延迟js文件在页面加载完成之后再加载，但是只有IE支持
  • async：异步加载和执行脚本，不会因为脚本的加载阻塞页面的加载

十、性能优化方法

  • 减少http请求次数：CSS精灵，JS、CSS源码压缩，图片大小控制合适，cdn托管，data缓存
  • innerHTML代替DOM操作
  • 图片预加载，将样式表放在顶部，脚本放在底部
  • 减少iframe数量
  • 避免404
